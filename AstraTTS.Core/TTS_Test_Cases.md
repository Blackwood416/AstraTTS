# TTS 中英混合测试用例

## 基础功能测试

| ID | 输入 | 预期效果 |
|----|------|---------|
| B1 | `你好世界` | 纯中文，正常发音 |
| B2 | `Hello world` | 纯英文，正常发音 |
| B3 | `我是AI` | 中英混合，无明显停顿 |

---

## 边界条件测试

### 语言切换

| ID | 输入 | 难点 |
|----|------|------|
| L1 | `这个feature很不错` | 英文在句中，后续内容不应被截断 |
| L2 | `AI是未来的趋势` | 英文在句首 |
| L3 | `我用Python写代码，然后用JavaScript做前端` | 多次中英切换 |
| L4 | `OpenAI发布了GPT4` | 多个英文片段连续 |

### 缩写与大写

| ID | 输入 | 难点 |
|----|------|------|
| A1 | `TTS` | 三字母缩写，OOV拼读 |
| A2 | `AI` | 两字母缩写 |
| A3 | `HTTP协议` | 四字母缩写 + 中文 |
| A4 | `我使用API调用LLM` | 多个缩写 |
| A5 | `ABCDEFG` | 长缩写 |

### 数字处理

| ID | 输入 | 难点 |
|----|------|------|
| N1 | `今天是2024年1月15日` | 年月日 |
| N2 | `价格是99.9元` | 小数 |
| N3 | `成功率达到95%` | 百分比 |
| N4 | `第1名` | 序数词 |
| N5 | `123456789` | 大数字 |
| N6 | `3.14159` | 长小数 |

### 特殊字符与边界

| ID | 输入 | 难点 |
|----|------|------|
| S1 | `A` | 单字母 (极短输入) |
| S2 | `I` | 单字母代词 |
| S3 | `OK!` | 英文 + 标点 |
| S4 | `Hello，世界` | 中文逗号 |
| S5 | `AI?` | 英文 + 问号 |
| S6 | `……` | 纯省略号 |
| S7 | ` ` | 空格 |
| S8 | `` | 空字符串 |

### 复杂混合场景

| ID | 输入 | 难点 |
|----|------|------|
| C1 | `用PyTorch训练了一个BERT模型，效果比TensorFlow好` | 多个技术术语 |
| C2 | `iPhone15Pro发布于2023年9月` | 产品名 + 日期 |
| C3 | `在GitHub上star了100个repo` | 英文动词+名词 |
| C4 | `C++和C#都是编程语言` | 含特殊字符的语言名 |
| C5 | `email地址是test@example.com` | 邮箱格式 |
| C6 | `访问https://www.baidu.com` | URL |

### 极端/压力测试

| ID | 输入 | 难点 |
|----|------|------|
| E1 | `AAAAAAAAAA` | 重复字母 |
| E2 | `这个这个这个feature feature feature` | 重复词 |
| E3 | `中English中English中` | 快速切换 |
| E4 | `abc123def456` | 数字字母混合 |
| E5 | 一段 200+ 字的长文本 | 长文本稳定性 |

- [ ] URL 和邮箱格式可能不会正确发音
